命令相关：

ip a：查看网卡
主要看的几点：一般来说网卡都是ens33，inet后面就是你的ipv4地址，/24表示子网掩码

ip route：查看路由
一般来说，default就是默认路由，以后上不了网了优先看默认路由还在不在

ping：看网络是否连通

ss -tnlp：查看监听端口

ss -tan：查看网络连接

nc：测试端口（好用）
命令示例：nc -zv 192.168.227.133 22

lsof：查看端口属于哪个进程
示例：lsof  -i：80

网络排查流程：
ip a查看ip是否正常，ip route查看默认网关是否存在，ping网关查看局域网是否正常，ping 8.8.8.8看是否能走出公网，ping baidu.com看DNS是否正常，ss -tnlp看服务是否监听端口
最后看firewall/安全组是否放行端口，nginx也就是服务本身是否正常

TCP专栏：

三次握手：第一次客户端发送SYN，表示想要建立连接，第二次服务端发送SYN加ACK，表示同意建立连接，第三次客户端回复ACK，表示收到，最后状态为ESTAB

四次挥手：第一次客户端发送FIN，第二次服务器收到了关闭请求，回复ACK，但是这个时候可能还有数据没传完，所以连接没有完全关闭，第三次服务器传输完成，发送FIN，最后第四次，客户端回复ACK，确认关闭

两个状态，CLOSE_WAIT和TIME_WAIT
TIME_WAIT：由主动关闭连接的一方产生，作用就是为了防止最后一次ACK丢失和防止旧连接的数据影响新的连接，产生之后，端口会进入一段时间的冷却，如果大量TIME_WAIT产生，就会导致主动关闭方的端口枯竭，客户端无法申请服务

CLOSE_WAIT：由被动关闭方产生，由于没有即使调用CLOSE()产生，危害特别大，会耗尽FD（文件描述符）和内存，通常要去修改代码

FD：**操作系统给每个打开的资源发的一个“号码牌”**。
